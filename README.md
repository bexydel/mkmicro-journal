# mkmicro-journal

一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对特定域名或网站，仅作为静态 HTML 内容的存储与展示空间。

## 项目简介

mkmicro-journal 旨在提供一种简洁的方式，将多个独立的 HTML 页面集中存放，并通过 GitHub Pages 或类似服务进行发布。每个页面均保持独立，互不依赖，适合用于记录、展示或归档轻量级内容。

## 目录说明

```
.
├── index.html          # 主页（如有）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、图片、字体等）
│   ├── css/
│   ├── images/
│   └── ...
└── README.md           # 本文件
```

- `pages/`：主要页面存放目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面间共享的静态资源，可按需组织子目录。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可直接在浏览器中打开。
- 页面之间无强制关联，可根据需要自由创建或删除。
- 资源文件建议使用相对路径引用，以保持仓库结构清晰。
- 本仓库不限制页面内容类型，但建议保持轻量与简洁。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 `.html` 文件，并确保引用资源路径正确。
- 更新页面：直接编辑对应 HTML 文件，提交后自动更新。
- 删除页面：移除对应文件，并检查是否有其他页面引用其资源。
- 资源管理：公共资源放入 `assets/` 目录，避免在页面内重复存放。

## 使用建议

- 可配合 GitHub Pages 启用静态站点，将仓库内容发布为网站。
- 如需自定义域名或站点配置，请参考 GitHub Pages 官方文档。
- 本仓库适合存放小型、独立、无需数据库的页面内容。
